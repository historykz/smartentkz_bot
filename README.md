# 🎓 ENT Bot — Telegram-бот для подготовки к ЕНТ

Многофункциональный Telegram-бот для подготовки к ЕНТ (Казахстан): тесты, пробники, викторины, конспекты, Daily ENT, дуэли 1×1, рейтинги, реферальная система, премиум.

Стек: **Python 3.10+**, **aiogram 3.x**, **SQLite** (без SQLAlchemy), **FSM**, **InlineKeyboard**, **asyncio**.

---

## 📦 Установка

### 1. Зависимости

```bash
git clone <repo> ent_bot
cd ent_bot
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Получение токена

1. В Telegram откройте [@BotFather](https://t.me/BotFather)
2. `/newbot` → задайте имя и username
3. Скопируйте токен (вида `123456:ABC-DEF...`)

### 3. Конфигурация (`.env`)

Создайте файл `.env` в корне проекта:

```ini
BOT_TOKEN=123456:ABC-DEF...
ADMIN_IDS=111222333,444555666
MANAGER_USERNAME=historyentk_bot
DB_PATH=ent_bot.db
LOG_LEVEL=INFO
LOG_FILE=bot.log
```

- `BOT_TOKEN` — токен от BotFather (обязательно)
- `ADMIN_IDS` — Telegram ID администраторов через запятую. Свой ID можно узнать у [@userinfobot](https://t.me/userinfobot)
- `MANAGER_USERNAME` — username менеджера (без `@`) для платных тестов и техподдержки
- `DB_PATH` — путь к файлу SQLite (создаётся автоматически)

### 4. Запуск

```bash
python main.py
```

База данных создаётся автоматически при первом запуске.

## ⚙️ Настройка бота в BotFather

После создания бота откройте [@BotFather](https://t.me/BotFather) и выполните:

1. `/mybots` → выбрать бота → **Bot Settings**
2. **Group Privacy** → **Turn off** (чтобы бот видел сообщения в группах для викторин)
3. **Allow Groups?** → **Turn on**
4. Вернуться → **Inline Mode** → **Turn on** (для поиска тестов через `@botname ...`)

### Каналы для обязательной подписки

Если в боте включены «обязательные каналы», **добавьте бота администратором** в эти каналы (с правом «Просмотр участников»), иначе проверка подписки не сработает.

## 👤 Использование

### Для пользователей

- `/start` — главное меню
- `/menu` — открыть меню в любой момент
- `/help` — помощь
- `/cancel` — прервать текущее действие

**Возможности:**
- 📚 Каталог тестов с фильтром по языку
- 📝 Пробники ЕНТ
- 🧠 Викторины (в т.ч. групповые: `/quiz <ID_теста>` в группе)
- 📖 Конспекты с постраничным чтением, защитой от пересылки
- 📅 Daily ENT — ежедневный тест со streak'ами
- ⚔️ Дуэли 1 на 1 (быстрый поиск соперника)
- 🏆 Рейтинги: общий / неделя / Daily / школы
- 🎁 Реферальная система
- 👑 Premium

### Inline-режим (для шеринга)

В любом чате наберите:
```
@your_bot_name алгебра
```
Бот покажет список подходящих тестов. Тап по результату — отправляет красивую карточку с deep-link.

### Deep-link'и

- `https://t.me/your_bot?start=test_42` — открыть карточку теста №42
- `https://t.me/your_bot?start=note_15` — открыть конспект №15
- `https://t.me/your_bot?start=ref_123456` — реферальная ссылка от пользователя 123456

## 🛠 Админ-панель

Откройте бот в личном чате, отправьте `/admin` (доступно только если ваш `tg_id` указан в `ADMIN_IDS`).

### Возможности

| Кнопка | Что делает |
|---|---|
| 📝 Создать тест | Пошаговый мастер (название → описание → предмет → класс → язык → тип → таймер → лимит попыток → платный?  → перемешивание → подписка на канал) |
| 📚 Мои тесты | Список всех тестов с действиями: импорт, вопросы, удаление |
| 📥 Импорт текстом | Массовая загрузка вопросов из текста |
| 🗳 Импорт Quiz Poll | Пересылка Telegram Quiz Polls |
| 📋 Черновики | Опросы без указанного правильного ответа — выбираете вручную |
| 🎟 Выдать доступ | Платный тест → конкретному пользователю |
| 👑 Premium | Выдача Premium на N дней (0 = бессрочно) |
| 🚫 Блок | Заблокировать/разблокировать пользователя |
| 📢 Каналы | Управление обязательной подпиской |
| 📖 Конспекты | Создание новых конспектов, удаление |
| 📊 Статистика | Сводка по пользователям, тестам, попыткам |
| 📤 Экспорт CSV | Выгрузка таблицы пользователей |

### Импорт вопросов (текстом)

Каждый вопрос отделяется **пустой строкой**. Правильный ответ помечается `*` в конце варианта:

```
Столица Казахстана?
A) Алматы
B) Шымкент
C) Астана *

Сколько будет 7 × 8?
A) 54
B) 56 *
C) 64
```

Поддерживаются буквы A–J (латиница) и А–К (кириллица). Один вопрос — один правильный ответ.

### Импорт Quiz Poll

1. В админ-меню → 🗳 Импорт Quiz Poll
2. Выберите тест
3. **Перешлите** в чат сообщение типа Quiz Poll (с указанным правильным ответом)
4. Если правильный ответ в Poll не указан — попадёт в **📋 Черновики**, где вы вручную выберете правильный вариант

### Выдача Premium

Premium снимает ограничения по количеству попыток, открывает Premium-конспекты.

`/admin` → 👑 Premium → ID пользователя или @username → количество дней (0 = бессрочно).

## 🔒 Защита контента

В боте включены следующие меры:

- **`protect_content=True`** — Telegram запрещает пересылку и копирование сообщений с вопросами и конспектами (на уровне клиента). От скриншотов **не защищает** — Bot API такой возможности не предоставляет.
- **Перемешивание вариантов и вопросов** — за попытку сохраняется фиксированный порядок в БД, чтобы нельзя было «подсмотреть» через старые кнопки.
- **Server-side валидация ответов** — callback хранит только `option_id`, без флага «правильный/нет». Подделка callback не работает.
- **Anti-spam** — middleware ограничивает частоту нажатий (0.5 сек минимум).
- **Лимит попыток** — задаётся в настройках теста.
- **`first_attempt_only`** — для учёта в рейтинге засчитывается только первая попытка.

## 🐛 Troubleshooting

### Бот не отвечает
- Проверьте `BOT_TOKEN` в `.env` (без кавычек, без пробелов)
- Логи: `tail -f bot.log`

### Не работает в группах
- В BotFather: **Bot Settings → Group Privacy → Turn off**
- Бот должен быть **добавлен в группу как участник** (для викторин — желательно как админ)

### Не открываются Inline-результаты
- BotFather: **Bot Settings → Inline Mode → Turn on**
- Установите inline placeholder: `/setinlinefeedback` (опционально)

### Проверка подписки не работает
- Бот должен быть **админом в канале** обязательной подписки

### Платный тест не открывается после оплаты
- В админ-панели: **🎟 Выдать доступ** → указать пользователя и ID теста

### База данных «битая»
- Остановите бота, удалите `ent_bot.db`, перезапустите — БД пересоздастся пустой
- Данные пользователей и тесты при этом потеряются

## 📊 Архитектурные решения

- **Без SQLAlchemy** — прямые SQL-запросы через `sqlite3` (быстрее, проще, меньше зависимостей)
- **In-memory state для активных тестов/дуэлей** — таймеры через `asyncio.create_task()`. При перезапуске бота активные тесты прерываются (это допустимо: пользователь увидит «возобновить?»).
- **Группа FSM States по доменам** — `TestCreateStates`, `NoteCreateStates`, `DuelStates` и т.д.
- **Хендлеры разнесены по файлам** — каждый домен в своём router'е, регистрация в `main.py`.
- **Локализация — словарь** — `locales.TEXTS[key][lang]`, fallback на `ru`.
